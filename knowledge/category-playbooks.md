## CAT-01: cosmetics（化粧品・美容液・スキンケア）

```
pattern_id        : CAT-01
category          : cosmetics
typical_personas  : pain-avoider, lifestyle-fit, quality-first, aspiration-driven
price_band_range  : low〜premium（商材によって幅広い）
key_decision_factor: 成分・口コミ・ブランドの世界観・安心感
legal_risk        : 薬機法・景表法（効果の断定表現に注意）
```

### ターゲット解像度

```
悩み    : 乾燥・くすみ・ニキビ跡・毛穴・年齢による変化
心理    : 「また違う商品にお金をかけるのが怖い」「本当に効くか不安」
動機    : 毎日使うものだから、続けやすくて自分に合うものが欲しい
        : 特別な日ではなく、日常の延長として選びたい
```

### 訴求軸の優先順

```
1. 体験・感覚（使った後の肌の変化。五感に訴える）
2. 安心感（成分・肌タイプへの配慮・敏感肌対応）
3. 世界観（ブランドの美意識・日常への溶け込み）
4. 価格・コスパ（price_band が mid 以下の場合のみ前面に出す）
```

### 推奨パターン組み合わせ

```
structure_template: story（第1推奨）/ trust（成分訴求が主軸の場合）
hook             : HOOK-02（共感型・最重要）/ HOOK-01（問いかけ型）
cta_style        : soft（lifestyle-fit, quality-first の場合）
                 : bold（pain-avoider で解決を急いでいる場合）
tone             : TONE-COMBO-01（clean × premium、高価格帯）
                 : TONE-COMBO-04（clean × warm、中価格帯）
                 : TONE-07（reassuring を必ず組み合わせる）
```

### コピーライティングの重点

```
○ 「肌の印象が変わった気がした」（主観・非断定で体験を描写）
○ テクスチャ・香り・使い心地などの感覚描写を豊かに
○ 「忙しい人でも」「朝でも夜でも」などの生活密着描写
○ 敏感肌への配慮を一言添える
```

### 避けるべき表現（法的リスク）

```
× 「効果が出ます」「必ず改善」「シミが消える」
× 医薬品的な表現（「治療」「治す」「改善する」）
× 過度な比較（他社商品との優劣）
× 「激変」「劇的変化」など誇大な程度表現
```

### デザインのヒント

```
- 余白を広めに取り、清潔感を演出（section_spacing: large 推奨）
- main_color には淡い色（ピンク・アイボリー・ラベンダー）が馴染みやすい
- 商品ビジュアルの代替は「柔らかいグラデーション」
- font_style: serif 推奨（高価格帯）
```

## CAT-02: supplement（サプリメント・健康食品・機能性食品）

```
pattern_id        : CAT-02
category          : supplement
typical_personas  : skeptic, result-seeker, pain-avoider
price_band_range  : low〜high
key_decision_factor: 成分の根拠・継続しやすさ・価格・返金保証
legal_risk        : 機能性表示食品の規定・薬機法・景表法
```

### ターゲット解像度

```
悩み    : 疲れが取れない・集中できない・体型・免疫力・特定の栄養不足
心理    : 「本当に効くの？」「続けないと意味ないんじゃ」「飲みにくくないか」
動機    : 手軽に健康管理をしたい。食事だけでは補えない何かを取りたい
```

### 訴求軸の優先順

```
1. 根拠・信頼（成分・研究・専門家の視点）
2. 続けやすさ（飲みやすさ・味・習慣化しやすい形状）
3. 実感（他の人がどう感じたか・時系列のレビュー）
4. 価格・コスパ
```

### 推奨パターン組み合わせ

```
structure_template: trust（第1推奨）/ story（感情訴求が主軸の場合）
hook             : HOOK-01（問いかけ型・最重要）/ HOOK-03（数字型・根拠がある場合）
cta_style        : soft または bold（skeptic には soft から始める）
tone             : TONE-COMBO-02（warm × reassuring・最重要）
                 : TONE-07（reassuring を必ず組み合わせる）
```

### コピーライティングの重点

```
○ 成分説明は「難しい名前 → 生活への影響」の順で展開する
○ レビューは「○週間後に○○が変わった気がした」という時系列表現が信頼感を高める
○ 「なぜ今まで変わらなかったか」の原因説明から始め、その後商品を提示する
```

### 避けるべき表現（法的リスク）

```
× 「病気を治す」「改善する」「医師が推薦」（根拠なし）
× 「飲むだけで○○になれる」などの断言
× 「○○の人全員に効果がある」
× ビフォーアフターの誇大表現
```

### デザインのヒント

```
- 信頼を示す配色（深緑・紺・白などの落ち着いた色）
- 成分・根拠セクションは視覚的に「丁寧に整理された」印象に
- FAQ は多め（5〜7問）、especially 副作用・飲み合わせ・妊娠中などの質問
```

## CAT-03: apparel（衣類・ファッション雑貨・小物）

```
pattern_id        : CAT-03
category          : apparel
typical_personas  : aspiration-driven, lifestyle-fit, quality-first, gift-giver
price_band_range  : low〜premium
key_decision_factor: ビジュアル・素材感・ブランドの世界観・SNS映え
legal_risk        : 低（サイズ・素材の正確な表記に注意）
```

### ターゲット解像度

```
悩み    : 着回しが難しい・似合うかわからない・品質が長続きしない
心理    : 「着たことのない自分になれるかも」「これを着ている自分を想像したい」
動機    : 自己表現・特別な日の演出・毎日のちょっとした喜び
```

### 訴求軸の優先順

```
1. 世界観・ビジュアル（「このブランドが好き」という感情的なつながり）
2. 素材・品質（長く使えること・肌触り・縫製）
3. 着回し・スタイリング（「どんな日に着られるか」の具体的なイメージ）
4. 価格（price_band が mid 以下の場合のみ）
```

### 推奨パターン組み合わせ

```
structure_template: story（第1推奨）/ offer（シーズンセール・新作発売時）
hook             : HOOK-05（宣言型・最重要）/ HOOK-04（逆説型）
cta_style        : soft（高価格帯・quality-first）/ bold（セール時）
tone             : TONE-COMBO-01（clean × premium、高価格帯）
                 : TONE-05（playful、カジュアル系）
```

### コピーライティングの重点

```
○ 商品説明より「これを着たときの自分の気持ち」を描写する
○ 「オールシーズン使える」「10年着られる」など時間軸で価値を伝える
○ 色・素材の感覚描写（「シルクのような滑らかさ」「日光に透けて見える薄さ」）
```

### 避けるべき表現

```
× 他ブランドとの品質比較
× 「誰にでも似合う」（押しつけに聞こえる）
× サイズ・体型への否定的な表現
```

### デザインのヒント

```
- ビジュアルが最重要。代替グラフィックはシルエット・シルク感を意識したグラデーション
- 余白を広く取り、テキストを少なめに（世界観で見せる）
- layout_width: narrow（高価格帯）または standard
```

## CAT-04: course（オンライン講座・教材・書籍・コーチング）

```
pattern_id        : CAT-04
category          : course
typical_personas  : result-seeker, aspiration-driven, skeptic
price_band_range  : mid〜premium
key_decision_factor: 実績・カリキュラムの具体性・受講生の変化・価格と価値のバランス
legal_risk        : 景表法（実績の誇大表現に注意）
```

### ターゲット解像度

```
悩み    : スキル不足・キャリアへの不安・時間がない中で成長したい
心理    : 「本当に変わるのか」「自分でもできるのか」「時間とお金を無駄にしたくない」
動機    : 具体的な成果・スキル習得・人生を変えるきっかけ
```

### 訴求軸の優先順

```
1. 結果・変化（受講後にどうなれるか。具体的に）
2. 実績・証拠（受講生の声・数字・実績）
3. 方法論の独自性（なぜ他の方法より効果的か）
4. 続けやすさ（時間・難易度・サポート体制）
```

### 推奨パターン組み合わせ

```
structure_template: trust（第1推奨）/ story（ターゲットの課題に深く共感する場合）
hook             : HOOK-01（問いかけ型・最重要）/ HOOK-04（逆説型）
cta_style        : bold（result-seeker 向け）/ soft（skeptic に信頼構築してから）
tone             : TONE-COMBO-03（energetic × bold、行動喚起重視）
                 : TONE-07（reassuring を組み合わせて不安解消）
```

### コピーライティングの重点

```
○ 「学べる内容」より「学んだ後の自分」から逆算して書く
○ 受講生の変化は「○ヶ月後」「○項目が改善」など時間と数字で具体化
○ 「どんな人に向いているか / 向いていないか」を明記すると信頼が上がる
```

### 避けるべき表現

```
× 「必ず○○になれる」「○%が合格」（根拠なしの断言）
× 焦りを過度に煽る（「今すぐ変わらないと人生が変わらない」系）
× 実績の誇大表現
```

### デザインのヒント

```
- 情報量が多い → layout_width: standard / wide 推奨
- カリキュラム・ステップを視覚的に整理（番号付きリスト・ステップ図）
- 受講生の声は「変化前→変化後」の対比が効く
```

## CAT-05: app（アプリ・デジタルサービス・SaaS・Webサービス）

```
pattern_id        : CAT-05
category          : app
typical_personas  : result-seeker, skeptic, pain-avoider
price_band_range  : low〜high（特に free trial モデルが多い）
key_decision_factor: 機能の明確さ・使い始めのハードルの低さ・価格（無料トライアル）
legal_risk        : 低（セキュリティ・プライバシーへの説明は必要）
```

### ターゲット解像度

```
悩み    : 作業が非効率・手作業が多い・ツールが使いこなせていない
心理    : 「本当に自分に使えるか」「慣れるまで大変そう」「他のツールと何が違うか」
動機    : 時短・自動化・品質向上・コスト削減
```

### 訴求軸の優先順

```
1. 機能・できること（何ができるか具体的に）
2. 使いやすさ（最初の一歩が簡単）
3. 時間・コスト削減（どれくらい楽になるか）
4. 実績（ユーザー数・継続率・レビュー）
```

### 推奨パターン組み合わせ

```
structure_template: offer（第1推奨・無料トライアルをオファーとして使う）/ trust（実績訴求が主軸の場合）
hook             : HOOK-04（逆説型・最重要）/ HOOK-03（数字型）
cta_style        : bold（result-seeker 向け）/ urgent（トライアル期限がある場合）
tone             : TONE-COMBO-03（energetic × bold）
                 : TONE-01（clean、シンプルなプロダクトの場合）
```

### コピーライティングの重点

```
○ 機能名より「その機能で何が変わるか」を先に書く
○ 「○分でセットアップ完了」「今日から使える」などの即時性
○ ターゲットが今使っているツール・方法との比較を自然に入れる（過度な比較は避ける）
```

### 避けるべき表現

```
× 「業界最高」「唯一無二」（根拠が必要）
× 専門用語の多用（初心者向けの場合）
× セキュリティ・プライバシーへの不安を放置
```

### デザインのヒント

```
- スクリーンショット代替: UI の雰囲気を示す CSS グラフィック（グリッド・モックアップ）
- 機能リストは番号付きカードで視覚整理
- CTA は「無料で試す」「30日間無料」など lp_goal: trial を前面に
- layout_width: wide 推奨
```

## CAT-06: food（食品・飲料・菓子・ギフト食品）

```
pattern_id        : CAT-06
category          : food
typical_personas  : aspiration-driven, gift-giver, lifestyle-fit
price_band_range  : low〜premium
key_decision_factor: ビジュアル・味の魅力（コピーによる）・贈り物としての品格・価格帯
legal_risk        : 誇大広告（「最高においしい」などの根拠なし断言）・アレルギー情報の記載義務
```

### ターゲット解像度

```
悩み    : 「ちょっといいもの」を選ぶ基準がわからない・贈り物にふさわしいかわからない
心理    : 「食べる前からテンションを上げたい」「贈った相手に喜んでほしい」
動機    : 自分へのご褒美・大切な人へのギフト・特別な日の演出
```

### 訴求軸の優先順

```
1. 味・体験（食べたときの感覚を言葉で伝える）
2. 贈り物価値（「もらった人が喜ぶ姿」を想像させる）
3. 素材・製法（なぜこの味が実現できるか）
4. ブランドの哲学・世界観
```

### 推奨パターン組み合わせ

```
structure_template: offer（第1推奨・ギフトシーズン）/ story（ブランドストーリーが強い場合）
hook             : HOOK-05（宣言型・最重要）/ HOOK-02（共感型・ご褒美訴求）
cta_style        : soft（premium 価格帯）/ bold（ギフトシーズンのセール時）
tone             : TONE-COMBO-01（clean × premium、高価格帯）
                 : TONE-03（warm、カジュアルなギフト・日常食品）
```

### コピーライティングの重点

```
○ 五感をフル活用した描写（香り・食感・見た目・余韻）
○ 「もらった人の反応」を想像させる文章（ギフト訴求の場合）
○ 食べるシーン・時間帯・誰と食べるかのシチュエーション描写
```

### 避けるべき表現

```
× 健康効果の断言（「体に良い」「カロリーゼロなので太らない」）
× 「最高においしい」などの根拠なし断言（誇大広告に抵触するリスク）
× アレルギー情報の省略（必ずフッターに記載する）
```

### デザインのヒント

```
- 暖色系・素材感のある背景（テクスチャ表現）
- 商品ビジュアルの代替は「食材の色・質感を反映したグラデーション」
- SNS タイルは暖色系・食欲をそそる色調
```
