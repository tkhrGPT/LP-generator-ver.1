# LP ジェネレーター — 設計仕様書

このドキュメントは、LPジェネレーターの設計思想・役割分担・拡張方針をまとめたものです。
開発者向けのリファレンスです。エンドユーザー向けの情報は README.md を参照してください。

---

## このツールの思想

**「デザインの自由度」よりも「商材の差し替えやすさ」と「勝ちパターンの再利用」を優先する。**

LPを作る際の難しさは、ゼロから考えることではなく「毎回同じクオリティを安定して出せること」にあります。
このツールは、エンドユーザーの入力情報から Claude がペルソナを推定し、
knowledge パターンライブラリから最適なフック・CTA・構成・トーンを選定して生成することで、
異なる商材・世界観・訴求に対応しながら LP として機能する品質を維持します。

---

## 役割分担

```
エンドユーザーが決めること     開発者が管理すること（knowledge）  Claude が判断すること
──────────────────────       ────────────────────────────────   ──────────────────────────────
・ブランド名・商品名           ・フックパターンと選定基準           ・ペルソナ仮説の構築
・色（3色）                   ・CTAパターンと配置ルール            ・コピーライティング（文章生成）
・デザイントーン（選択式）      ・セクションパターンの設計指針       ・HTMLの構造・セクション内容
・LP構成テンプレート（選択式）  ・トーンパターン別の文体・デザイン指針 ・CSSのレイアウト詳細
・商品の説明・FAQ・レビュー    ・カテゴリ別の訴求戦略と避けるべき表現 ・knowledge からのパターン選定
・オファー内容                                                   ・選定理由の明記
・フォーム項目                                                   ・legal_notes に準拠した言い換え
・法的注意事項
```

**4者の関係**
- エンドユーザー: 商材固有の情報を提供する（何を・誰に・どんな世界観で）
- 開発者: knowledge パターンライブラリを整備・改善する（汎用的な勝ちパターン）
- knowledge: Claude が参照する「内部パターンライブラリ」として機能する
- Claude: ユーザーの情報からペルソナを推定し、knowledge のパターンを選定して今回向けに生成する

Claude は「ペルソナを推定 → パターンを選ぶ → HTML/CSS/コピーに変換するエンジン」として機能します。
エンドユーザーは「値を選ぶ・埋める」だけで、knowledge や PROMPT.md を触る必要はありません。

## knowledge フォルダの位置づけ

knowledge フォルダは**開発者が管理する内部パターンライブラリ**です。

```
エンドユーザーが触る層         開発者が管理する層（内部）         自動生成
──────────────────────       ────────────────────────────────   ──────────────────────────────
input/brand-vars.json  →     knowledge/copy-hooks.md            output/index.html
input/content-input.md →     knowledge/cta-patterns.md
                             knowledge/section-patterns.md
                             knowledge/tone-patterns.md
                             knowledge/category-playbooks.md
                    ↓        ↑
                    Claude が読み込み、ペルソナを推定し、パターンを選定する
```

**エンドユーザーは knowledge を触りません。**
knowledge の改善は開発者が行い、LP生成品質の向上に直接反映されます。

## knowledge パターンの構造

各 knowledge ファイルのパターンは以下のメタデータを持ちます:

```
pattern_id         : ユニークな識別子（例: HOOK-01, CTA-S01）
pattern_name       : パターンの名称
summary            : 1文の概要
suitable_categories: 合うカテゴリ（CAT identifier）
suitable_personas  : 合うペルソナ（vocabulary から）
suitable_price_band: 合う価格帯（low/mid/high/premium）
suitable_lp_goals  : 合う LP ゴール（purchase/trial/inquiry/signup）
suitable_templates : 合う構成テンプレート（story/trust/offer）
tone               : 合うトーン
example_lines      : 抽象化されたオリジナル例文（実在ブランドの文言は使わない）
avoid_when         : このパターンを使わない条件
```

---

## ペルソナ vocabulary（全パターン共通の語彙）

```
pain-avoider       : 悩みから逃げたい（現状への不満が動機）
result-seeker      : 具体的な変化・成果が欲しい
aspiration-driven  : 理想の自分・生活を目指している
skeptic            : 効果に懐疑的、証拠を求める
gift-giver         : 誰かへのプレゼントとして買う
lifestyle-fit      : 日常に無理なく溶け込むものが欲しい
quality-first      : 素材・製法・ブランドの哲学に価値を置く
```

## 固定部分と可変部分

### 固定（変えない・変えるべきでない）

| 固定要素 | 理由 |
|---|---|
| セクション構成の大枠（ヒーロー・特徴・FAQ・フォーム・フッター） | どんな商材でも必要な最低構成 |
| HTML単体ファイルで完結する形式 | 非エンジニアがファイルをブラウザで開くだけで確認できるようにするため |
| 外部ライブラリ不使用 | CDNへの依存をなくすため |
| CTAボタンの複数配置（4〜5箇所） | CVRのための必須要件 |
| モバイルファースト設計 | toC商材のLPはスマホ流入が前提 |
| legal_notes の遵守 | 薬機法・景表法など |
| IMAGE-PLACEHOLDER コメント（3箇所） | 画像差し替えを非エンジニアが行えるようにするため |

### 可変（入力変数で切り替えられる）

入力変数の詳細は `input/brand-vars.json` および README.md を参照。

## 改善サイクルの回し方

### 短期（生成のたびに）

1. LP を確認し「何が違うか」を言語化する
2. `brand-vars.json` の変数（`brand_tone` `structure_template` 等）を修正する
3. `content-input.md` のテキストを磨く
4. 再生成して比較する

### 中期（複数商材を扱う中で）

1. うまくいったパターンの `brand-vars.json` を別名保存しておく
2. knowledge のパターンに「この組み合わせが効いた理由」をコメントとして追記する
3. 新しいフックパターン・CTAパターンを `knowledge/` に追加する

### 長期（knowledge を進化させる）

- **パターンを増やす**: 新しいカテゴリ・ペルソナへの対応パターンを追加する
- **パターンを精緻化する**: 生成結果に一貫して問題が出るときは、該当パターンの `avoid_when` を更新する
- **ペルソナ vocabulary を拡張する**: 既存の7語では表現できないペルソナが出てきたら追加する

## ファイルごとの変更頻度の目安

| ファイル | 変更頻度 | 変更する人 |
|---|---|---|
| `input/brand-vars.json` | 商材ごとに毎回 | エンドユーザー |
| `input/content-input.md` | 商材ごとに毎回 | エンドユーザー |
| `knowledge/*.md` | 品質改善のときに | 開発者のみ |
| `prompt/PROMPT.md` | 生成フローを変えたいときだけ | 開発者のみ |
| `generator-spec.md` | ツールの方針が変わったときだけ | 開発者 |
| `README.md` | 手順・選択肢が増えたときだけ | 開発者 |
| `output/index.html` | 触らない（自動上書き） | Claude Code |
